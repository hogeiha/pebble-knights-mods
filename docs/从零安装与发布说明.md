# Pebble Knights Mod 从零安装与发布说明

这份文档用于把 **Mod 加载器** 和 **Mod 本体** 分开说明，方便玩家从刚安装好的 Steam 游戏开始安装，也方便你后续整理发布包。

## 1. 组件关系

这个 mod 体系由 BepInEx、加载器和独立 mod 文件夹组成：

```text
第 0 层：BepInEx
  作用：让 Unity 游戏启动时可以加载 C# 插件。
  来源：第三方 Unity mod 框架。

第 1 层：PebbleKnights.ModLoader
  作用：读取游戏根目录 Mods/<ModName>/manifest.json，并加载各 mod 的 DLL。
  文件：PebbleKnights.ModLoader.dll

第 2 层：CustomTraitFilter
  作用：特性自定义 mod，按 F8 打开特性启用/禁用面板。
  文件：CustomTraitFilter.dll + manifest.json + trait-filter.json

第 2 层：UniversalKingActions
  作用：让所有玩家可以使用国王式抱起/投掷与特性升级行为；真人骑士自己选择特性升级，AI/bot 由国王选择；帐篷升级和奖励保留，额外的技能/特性升级篝火入口隐藏。
  文件：UniversalKingActions.dll + manifest.json
```

玩家最终看到的安装结构应该是：

```text
Pebble Knights/
  Pebble Knights.exe
  UnityPlayer.dll
  Pebble Knights_Data/

  BepInEx/
    core/
    plugins/
      PebbleKnights.ModLoader.dll

  doorstop_config.ini
  winhttp.dll

  Mods/
    CustomTraitFilter/
      manifest.json
      plugins/
        CustomTraitFilter.dll
      config/
        trait-filter.json
    UniversalKingActions/
      manifest.json
      plugins/
        UniversalKingActions.dll
```

## 2. 从 Steam 找到游戏根目录

玩家刚从 Steam 安装好游戏后，先找到游戏根目录。

Steam 中操作：

```text
库 -> Pebble Knights -> 右键 -> 管理 -> 浏览本地文件
```

打开后应能看到这些文件：

```text
Pebble Knights.exe
UnityPlayer.dll
Pebble Knights_Data/
```

如果看不到 `Pebble Knights.exe`，说明当前不是游戏根目录。

默认路径通常是：

```text
C:\Program Files (x86)\Steam\steamapps\common\Pebble Knights
```

后文所有“游戏根目录”都指这个文件夹。

## 3. 玩家安装步骤

### 第一步：安装 BepInEx

这个游戏是 Unity Mono x64，所以需要 Windows x64 Mono 版 BepInEx 5。

玩家需要下载：

```text
BepInEx_win_x64_5.x.x.x.zip
```

不要下载：

```text
BepInEx_win_x86
BepInEx_linux_x64
BepInEx IL2CPP 版本
```

解压后，把压缩包根目录里的内容全部复制到游戏根目录。

复制后游戏根目录应该出现：

```text
BepInEx/
doorstop_config.ini
winhttp.dll
.doorstop_version
```

注意：不是只复制 `BepInEx/` 文件夹，`winhttp.dll` 和 `doorstop_config.ini` 也必须在 `Pebble Knights.exe` 同级。

然后启动游戏一次，确认生成：

```text
BepInEx/config/
BepInEx/plugins/
BepInEx/LogOutput.log
```

看到这些文件夹和日志，说明 BepInEx 已经生效。

### 第二步：安装 Mod 加载器

把你的加载器文件复制到：

```text
游戏根目录/BepInEx/plugins/PebbleKnights.ModLoader.dll
```

最终位置：

```text
Pebble Knights/
  BepInEx/
    plugins/
      PebbleKnights.ModLoader.dll
```

这个文件只负责加载 `Mods` 文件夹里的 mod，本身不包含具体玩法功能。

### 第三步：安装特性自定义 Mod

把 `CustomTraitFilter` 整个文件夹复制到游戏根目录的 `Mods` 文件夹里。

最终位置：

```text
Pebble Knights/
  Mods/
    CustomTraitFilter/
      manifest.json
      plugins/
        CustomTraitFilter.dll
      config/
        trait-filter.json
```

如果游戏根目录没有 `Mods` 文件夹，可以手动创建，或者让 ModLoader 第一次启动时自动创建。

### 第四步：安装全员国王动作 Mod

把 `UniversalKingActions` 整个文件夹复制到游戏根目录的 `Mods` 文件夹里。

最终位置：

```text
Pebble Knights/
  Mods/
    UniversalKingActions/
      manifest.json
      plugins/
        UniversalKingActions.dll
```

这个 mod 不需要游戏内菜单，启动后自动生效。

### 第五步：验证

启动游戏后打开：

```text
游戏根目录/BepInEx/LogOutput.log
```

正常日志应类似：

```text
Loading [Pebble Knights Mod Loader 0.1.0]
Started mod: Custom Trait Filter 0.1.0
Started mod: Universal King Actions 0.1.0
Loaded 120 traits. Trait filtering applies immediately to loaded data.
```

如果看到：

```text
Started mod: Custom Trait Filter 0.1.0
Started mod: Universal King Actions 0.1.0
```

说明两个 mod 都已加载。

## 4. 游戏内使用

进入游戏后：

```text
F8：打开/关闭特性自定义面板
F9：刷新特性列表
PageUp：上一页
PageDown：下一页
Home：第一页
End：最后一页
```

面板中可以：

```text
启用/禁用单个特性
全部启用
全部禁用
搜索特性
翻页查看特性
```

配置保存位置：

```text
游戏根目录/Mods/CustomTraitFilter/config/trait-filter.json
```

如果想恢复全启用，可以：

1. 在游戏里点 `全部启用`。
2. 或关闭游戏后把 `trait-filter.json` 改成：

```json
{
  "disabledTraits": []
}
```

`UniversalKingActions` 自动生效：

```text
所有玩家可以使用国王式抱起/投掷行为
特性升级相关逻辑按国王能力处理
真人骑士触发特性升级时由本人选择
AI/bot 触发特性升级时由国王选择
帐篷升级和奖励保留
额外的技能/特性升级篝火入口会被隐藏
```

## 7. 常见问题

### F8 没反应

检查：

```text
BepInEx/plugins/PebbleKnights.ModLoader.dll 是否存在
Mods/CustomTraitFilter/manifest.json 是否存在
Mods/CustomTraitFilter/plugins/CustomTraitFilter.dll 是否存在
Mods/UniversalKingActions/manifest.json 是否存在
Mods/UniversalKingActions/plugins/UniversalKingActions.dll 是否存在
```

再看：

```text
BepInEx/LogOutput.log
```

是否出现：

```text
Started mod: Custom Trait Filter 0.1.0
Started mod: Universal King Actions 0.1.0
```

### 游戏没有生成 BepInEx 日志

通常是 BepInEx 没装对。检查 `winhttp.dll` 和 `doorstop_config.ini` 是否和 `Pebble Knights.exe` 同级。

### 特性禁用后刷新消失

正式版已修复。列表使用第一次读取到的默认总列表，禁用后刷新仍会保留条目。

### 角色不能移动

本项目不会修改 `Cursor.lockState` 或 `Cursor.visible`。如果遇到输入异常，建议先关闭其他第三方 mod 并重启游戏确认。

## 8. 给其他开发者写 mod 的最小结构

一个普通 mod 文件夹应长这样：

```text
Mods/
  YourModName/
    manifest.json
    plugins/
      YourMod.dll
    config/
      your-config.json
```

`manifest.json` 示例：

```json
{
  "id": "your_mod_id",
  "name": "Your Mod Name",
  "version": "0.1.0",
  "author": "your name",
  "entry": "YourNamespace.YourModClass",
  "description": "Your mod description.",
  "enabled": true,
  "tags": ["example"]
}
```

C# 入口类需要实现：

```csharp
using PebbleKnights.ModLoader;

namespace YourNamespace
{
    public sealed class YourModClass : IPebbleMod
    {
        public void Load(IModContext context)
        {
            context.Logger.LogInfo("Your mod loaded.");
        }
    }
}
```

开发者需要引用：

```text
PebbleKnights.ModLoader.dll
BepInEx.dll
UnityEngine.dll
UnityEngine.CoreModule.dll
```

如果要读游戏数据，还需要根据功能引用：

```text
Scripts.dll
KBluePurple.Data.dll
```
